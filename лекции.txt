Основные тезисы лекции 2:
1) сновные команды Git
1. git init: Инициализирует новый репозиторий Git в текущей директории.
2. git clone: Создает локальную копию удаленного репозитория.
3. git add: Добавляет изменения в файлах.
4. git commit: Создает новый коммит с текущими изменениями.
5. git push: Отправляет локальные коммиты в удаленный репозиторий.
2)Виды слияния Git:
● git merge;
● git rebase - работаем с веткой самостоятельно и не планируем публиковать её на сервере.
Команда rebase запоминает коммиты из ветки в виде патчей, «перематывает» текущую ветку (как будто и не было никакого branch) и применяет патчи, 
оформляя их в виде коммитов. В отличие от rebase, merge делает слияние двух веток в одну.
3)Gitflow - методология работы с Git, определяет, какие ветки нужно создать и как производить их слияние.
